# 세니얼 최종본 - 이미지 프롬프트 쇼츠 생성기

## 핵심 방식
- 확장프로그램 없이 `index.html`만 실행합니다.
- 대본을 입력하거나 붙여넣고, 음성파일을 넣은 뒤 `장면 자동나누기`를 누르면 장면이 생성됩니다.
- `장면별 프롬프트 만들기`를 누르면 썸네일 프롬프트와 장면 프롬프트가 함께 생성됩니다.
- 썸네일 문구는 별도 합성이 아니라 이미지 프롬프트 안에 직접 들어갑니다.
- 선택한 이미지 스타일이 썸네일과 장면에 같이 적용됩니다.
- 완성 이미지는 각 카드에 드래그하거나 첨부 버튼으로 넣습니다.
- 영상은 소리 안정성을 위해 WEBM으로 생성합니다.

## 포함 기능
- 대본 복사/편집
- 음성파일 길이 기준 자막 싱크 적용
- 장면별 이미지 생성 프롬프트
- 썸네일 자동 만들기
- 좌측 이미지 스타일은 실사/만화 2개만 선택
- 실사/만화 선택 후 인물/배경/인물배경은 장면마다 자동 랜덤 적용
- 장면별 드래그 첨부
- 자막 위치/크기/색상 수정
- 미리보기
- WEBM 생성 및 다운로드
- 작업 저장/불러오기(JSON)

## 사용 순서
1. `index.html` 실행
2. 대본 입력 또는 붙여넣기
3. 음성파일 첨부 후 `음성 길이 적용` 클릭
4. `장면 자동나누기` 클릭
5. 이미지 스타일에서 실사 또는 만화 선택
6. `장면별 프롬프트 만들기` 클릭하면 인물/배경/인물배경이 장면별로 랜덤 적용
7. 맨 위 썸네일 카드에서 `썸네일 생성하기` 클릭
8. 생성된 썸네일 이미지를 카드에 드래그
9. 장면별 `생성하기` 클릭 후 이미지 드래그 첨부
10. `WEBM 생성 및 다운로드`

## 주의
브라우저 보안상 ChatGPT 입력창 자동 입력은 확장프로그램 없이 불가능합니다. 대신 프롬프트가 자동 복사되므로 생성창에서 붙여넣기만 하면 됩니다.


## V33 수정 - 재생/다운로드 안정화
- 정지 버튼을 누르지 않아도 재생 버튼만 다시 누르면 처음/현재 위치에서 바로 재생되도록 수정했습니다.
- 미리보기 종료 후 내부 재생 상태가 남아 재생 버튼이 먹지 않던 문제를 수정했습니다.
- 다운로드는 WEBM를 우선 시도하고, 브라우저가 WEBM 오디오를 제대로 지원하지 않으면 WEBM으로 자동 fallback 합니다.
- 브라우저 생성 방식 특성상 일부 크롬 환경에서는 안정적인 소리 보존을 위해 WEBM으로 저장될 수 있습니다.

## V40 수정 - 올자동 주제 우선 + 자막 싱크 + 이미지 프롬프트
- 올자동모드에서 입력한 주제/선택한 주제가 이미지 프롬프트에 항상 포함되도록 수정했습니다.
- API 키가 없어도 입력한 키워드 기준으로 주제 후보를 만들도록 보정했습니다.
- 주제 생성 시 이전 대본/장면/이미지 캐시를 끊어 기본 상식 주제가 섞이지 않도록 했습니다.
- 자막 싱크는 글자수 단순 분배 대신 한글 길이, 영어/숫자, 쉼표, 마침표, 짧은 문장 보정을 함께 계산하도록 수정했습니다.
- 음성 길이 적용 시에도 같은 싱크 엔진으로 다시 계산됩니다.

## V41 수정 - 자막 싱크 균등형 보정
- 기존 글자수 중심 싱크는 긴 문장 뒤부터 자막이 밀릴 수 있어 균등형 싱크로 변경했습니다.
- 전체 음성 길이를 자막 개수 기준으로 먼저 나누고, 문장 길이는 30%만 반영합니다.
- 음성 파일을 넣거나 `음성 길이 적용`을 누르면 같은 방식으로 다시 계산됩니다.
- 이미지 프롬프트는 선택 주제를 계속 우선 반영합니다.


## V42 수정 - 대본 만들기 제거 + 음성 기준 싱크 + 주제 프롬프트
- 우측 `대본 만들기` 버튼을 제거했습니다.
- 대본 편집 안내문을 직접 붙여넣기와 음성파일 기준 흐름으로 수정했습니다.
- 이미지 프롬프트에서 기본 카테고리 `상식`이 자동으로 섞이지 않도록 수정했습니다.
- 음성파일 적용 시 문장 길이 가중치 중심으로 자막 시간을 다시 계산하도록 수정했습니다.

## V46 수정 - 음성 무음구간 기준 자막 싱크 보정
- 음성 파일 첨부 시 Web Audio API로 무음구간을 분석합니다.
- 자막 시작은 음성보다 약간 늦게 표시하고, 끝은 짧게 유지해 앞부분 선출력과 뒤쪽 밀림을 줄였습니다.
- 문장 사이 무음구간에는 다음 자막이 너무 빨리 넘어가지 않도록 보정했습니다.
- `음성 길이 적용` 버튼을 누르면 기존 자막 시간표를 새 음성 기준으로 다시 계산합니다.
- index.html/style.css 구조는 변경하지 않고 app.js 싱크 계산부만 수정했습니다.

## V47 수정 - 싱크 안정형 재수정
- V46의 문장 사이 무음구간 직접 매칭 방식을 제거했습니다.
- 중간 빈 구간에서 마지막 자막이 반복 표시되던 문제를 수정했습니다.
- 음성 길이 적용 시 기존 자막칸 수를 유지하고 시간만 다시 배치합니다.
- 첫 자막은 약간 늦게 시작하고, 전체 자막은 음성 전체 길이에 맞춰 순서대로 배치합니다.
- HTML/CSS는 변경하지 않았습니다.

## V48 수정 - 무음 정지형 자막 싱크
- 음성 파형에서 말소리 구간을 분석합니다.
- 자막 시간은 말소리 구간에서만 진행되도록 수정했습니다.
- 무음 구간에서는 다음 자막으로 넘어가지 않고 현재 자막을 유지합니다.
- 다음 말소리가 시작되는 순간 다음 자막이 뜨도록 보정했습니다.
- 기존 장면, 이미지, 스타일 기능은 건드리지 않았습니다.

## V51 수정 - 음성·자막 분석 순서 적용
- `음성 길이 적용`을 `음성·자막 분석하기`로 변경했습니다.
- 음성파일을 넣어도 바로 장면 시간을 만들지 않고, 분석 버튼을 눌렀을 때만 자막/장면 시간을 계산합니다.
- 분석 진행률을 표시합니다: 음성 메타데이터 확인 → 파형 분석 → 무음구간 찾기 → 자막/말소리 매칭 → 완료.
- 장면 시간은 `자막 1개 = 장면 1개 = 음성 구간 1개` 기준으로 다시 생성됩니다.
- 말하기 전에는 자막이 먼저 뜨지 않도록 빈 구간의 마지막 자막 fallback을 제거했습니다.
- 무음구간에서는 이전 자막을 유지하고, 다음 말소리가 시작되는 지점에서 다음 자막으로 넘어가도록 보정했습니다.
- 올자동모드는 대본 생성 후 음성파일이 있으면 음성·자막 분석과 이미지 프롬프트 생성을 이어서 진행하고, 음성파일이 없으면 음성 제작 후 분석하라는 안내만 표시합니다.

## V52 수정 - 음성·자막 분석 버튼 위치 조정
- `음성·자막 분석하기` 버튼과 분석 진행률 박스를 우측 `음성파일 넣기` 바로 아래로 이동했습니다.
- 좌측 음성 영역의 중복 분석 버튼은 숨겼습니다.
- 싱크 계산 로직은 V51 그대로 유지하고, 위치/UI만 수정했습니다.

## V58 수정 - 장면 이미지 드래그 연결
- 장면 썸네일 이미지를 아래 장면의 빨간 `이미지 연결하기` 영역으로 끌어놓으면 같은 이미지를 공유합니다.
- 연결된 장면은 같은 이미지 번호를 사용하며, 움직임 효과도 같은 값으로 맞춰 이어지게 재생됩니다.
- 외부 이미지 파일을 기존 파란 드롭칸에 넣는 기능은 그대로 유지했습니다.
- 자막 시간, 음성 싱크, WEBM 렌더 로직은 변경하지 않았습니다.

## V59 수정 - 이미지 연결 드래그 안정화
- 장면 썸네일 이미지를 아래 장면의 `이미지 연결하기` 빨간 영역으로 끌어놓으면 이미지 슬롯이 연결됩니다.
- 브라우저에서 커스텀 드래그 데이터가 누락되는 경우를 대비해 내부 드래그 상태 백업을 추가했습니다.
- 연결된 장면은 같은 이미지와 같은 모션을 공유합니다.
- 자막 시간, 음성 싱크, 렌더/다운로드 로직은 변경하지 않았습니다.

## V60 수정 - 썸네일 이미지 드래그 연결 보강

- 장면 썸네일 이미지를 누른 채 아래 장면의 `이미지 연결하기` 영역에 놓으면 같은 이미지로 연결됩니다.
- 연결된 장면은 같은 이미지와 같은 움직임을 공유합니다.
- 자막 싱크, 시간 계산, 렌더/다운로드 로직은 변경하지 않았습니다.


## V61 수정 - Gemini / ChatGPT 장면별 생성 버튼
- 장면 카드의 `장면 생성하기`를 `Gemini 생성`과 `ChatGPT 생성` 버튼으로 분리했습니다.
- 각 버튼을 누르면 해당 장면 프롬프트가 자동 복사되고 선택한 생성 사이트가 열립니다.
- 생성 버튼 클릭은 사용자가 직접 합니다.
- 음성 싱크, 장면 연결, WEBM 다운로드 로직은 변경하지 않았습니다.
- 표기 오류 잘못된 온도쿠 표기를 정상 표기로 수정했습니다.


## V63 수정 - 썸네일/이미지 스타일 최종 정리
- 이미지 스타일 선택을 `실사 랜덤`, `만화 랜덤` 2개로 단순화했습니다.
- 실사/만화 선택 후 장면마다 인물, 배경, 인물배경이 자동 랜덤 적용됩니다.
- 썸네일 문구는 `감성`, `명언` 같은 카테고리명이 아니라 실제 주제 기반 클릭 문구로 생성됩니다.
- 썸네일 프롬프트에 이미지 생성 명령을 유지했습니다.


## V64 수정 - 좌측 이미지 스타일 즉시 적용
- 좌측 이미지 스타일에서 `실사 랜덤` 또는 `만화 랜덤`을 클릭하면 썸네일과 장면 프롬프트가 즉시 다시 생성됩니다.
- `장면별 프롬프트 만들기`를 다시 누르지 않아도 선택한 스타일이 오른쪽 장면 카드에 바로 반영됩니다.
- 랜덤은 생성할 때마다 인물/배경/인물배경 중 하나로 자동 배치됩니다.
