# 세니얼 최종본 - 이미지 프롬프트 쇼츠 생성기

## 핵심 방식
- 확장프로그램 없이 `index.html`만 실행합니다.
- 대본을 입력하거나 붙여넣고, 음성파일을 넣은 뒤 `장면 자동나누기`를 누르면 장면이 생성됩니다.
- `장면별 프롬프트 만들기`를 누르면 썸네일 프롬프트와 장면 프롬프트가 함께 생성됩니다.
- 썸네일 문구는 별도 합성이 아니라 이미지 프롬프트 안에 직접 들어갑니다.
- 선택한 이미지 스타일이 썸네일과 장면에 같이 적용됩니다.
- 완성 이미지는 각 카드에 드래그하거나 첨부 버튼으로 넣습니다.
- 영상은 소리 안정성을 위해 WEBM으로 생성합니다.

## 포함 기능
- 대본 복사/편집
- 음성파일 길이 기준 자막 싱크 적용
- 장면별 이미지 생성 프롬프트
- 썸네일 자동 만들기
- 실사/만화, 인물배경/인물/배경 선택
- 장면별 드래그 첨부
- 자막 위치/크기/색상 수정
- 미리보기
- WEBM 생성 및 다운로드
- 작업 저장/불러오기(JSON)

## 사용 순서
1. `index.html` 실행
2. 대본 입력 또는 붙여넣기
3. 음성파일 첨부 후 `음성 길이 적용` 클릭
4. `장면 자동나누기` 클릭
5. 이미지 스타일 선택
6. `장면별 프롬프트 만들기` 클릭
7. 맨 위 썸네일 카드에서 `썸네일 생성하기` 클릭
8. 생성된 썸네일 이미지를 카드에 드래그
9. 장면별 `생성하기` 클릭 후 이미지 드래그 첨부
10. `WEBM 생성 및 다운로드`

## 주의
브라우저 보안상 ChatGPT 입력창 자동 입력은 확장프로그램 없이 불가능합니다. 대신 프롬프트가 자동 복사되므로 생성창에서 붙여넣기만 하면 됩니다.


## V33 수정 - 재생/다운로드 안정화
- 정지 버튼을 누르지 않아도 재생 버튼만 다시 누르면 처음/현재 위치에서 바로 재생되도록 수정했습니다.
- 미리보기 종료 후 내부 재생 상태가 남아 재생 버튼이 먹지 않던 문제를 수정했습니다.
- 다운로드는 WEBM를 우선 시도하고, 브라우저가 WEBM 오디오를 제대로 지원하지 않으면 WEBM으로 자동 fallback 합니다.
- 브라우저 생성 방식 특성상 일부 크롬 환경에서는 안정적인 소리 보존을 위해 WEBM으로 저장될 수 있습니다.

## V40 수정 - 올자동 주제 우선 + 자막 싱크 + 이미지 프롬프트
- 올자동모드에서 입력한 주제/선택한 주제가 이미지 프롬프트에 항상 포함되도록 수정했습니다.
- API 키가 없어도 입력한 키워드 기준으로 주제 후보를 만들도록 보정했습니다.
- 주제 생성 시 이전 대본/장면/이미지 캐시를 끊어 기본 상식 주제가 섞이지 않도록 했습니다.
- 자막 싱크는 글자수 단순 분배 대신 한글 길이, 영어/숫자, 쉼표, 마침표, 짧은 문장 보정을 함께 계산하도록 수정했습니다.
- 음성 길이 적용 시에도 같은 싱크 엔진으로 다시 계산됩니다.

## V41 수정 - 자막 싱크 균등형 보정
- 기존 글자수 중심 싱크는 긴 문장 뒤부터 자막이 밀릴 수 있어 균등형 싱크로 변경했습니다.
- 전체 음성 길이를 자막 개수 기준으로 먼저 나누고, 문장 길이는 30%만 반영합니다.
- 음성 파일을 넣거나 `음성 길이 적용`을 누르면 같은 방식으로 다시 계산됩니다.
- 이미지 프롬프트는 선택 주제를 계속 우선 반영합니다.


## V42 수정 - 대본 만들기 제거 + 음성 기준 싱크 + 주제 프롬프트
- 우측 `대본 만들기` 버튼을 제거했습니다.
- 대본 편집 안내문을 직접 붙여넣기와 음성파일 기준 흐름으로 수정했습니다.
- 이미지 프롬프트에서 기본 카테고리 `상식`이 자동으로 섞이지 않도록 수정했습니다.
- 음성파일 적용 시 문장 길이 가중치 중심으로 자막 시간을 다시 계산하도록 수정했습니다.
