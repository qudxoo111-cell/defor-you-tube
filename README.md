# 세니얼 Senial - GitHub Pages 테스트용

## 업로드 파일
- index.html
- style.css
- app.js
- README.md

## GitHub Pages 사용법
1. 새 저장소 생성: `senial`
2. 위 4개 파일 업로드
3. Settings → Pages
4. Branch: `main` / root 선택
5. 생성된 주소로 테스트

## 포함 기능
- 올자동 모드
- 자막 직접 편집: 1줄/2줄/3줄 이상 자유 입력
- 이미지 배치: 자막 1칸/2칸/3칸/직접 설정
- 장면별 이미지 직접 업로드
- 장면별 AI 이미지 생성
- 썸네일 전용 생성기
- 기본 이미지 움직임: 확대 → 위로 → 아래로 반복
- 장면별 움직임 수정
- 자막 크기 조절
- 영상 빠르기 선택 및 직접 입력
- 브루형 중앙 미리보기

## 주의
브라우저에서 API 키를 직접 쓰는 구조는 테스트용입니다. 실제 서비스/판매용은 서버형으로 바꾸는 게 안전합니다.

## v1.6 수정 내용
- 장면 텀이 길게 잡히는 문제를 줄이기 위해 `자막/장면 촘촘히 나누기` 버튼을 추가했습니다.
- 분할 기준 초를 직접 입력할 수 있습니다. 기본값은 3.2초입니다.
- 각 장면마다 `이 장면 더 잘게 나누기` 버튼을 추가했습니다.
- 작업자가 직접 자막칸을 추가하고 시작/끝 시간을 수정할 수 있습니다.
- 촘촘히 나눌 때 자막 1칸당 이미지 1장으로 자동 변경하는 옵션을 추가했습니다.


## v1.7 수정 내용
- `자막/장면 촘촘히 나누기`를 누르면 자막만 쪼개지지 않고 이미지 장면도 같이 1:1로 분할됩니다.
- 기존 장면의 이미지/움직임/프롬프트를 새로 나뉜 장면들이 이어받습니다.
- 분할 후 각 자막칸마다 이미지를 따로 업로드하거나 AI 생성할 수 있습니다.
- `이 장면 더 잘게 나누기`도 동일하게 자막과 이미지 장면을 함께 나누도록 수정했습니다.

## v1.8 최종 수정 내용
- 한 컷 자막 글자 수를 기존보다 절반 수준으로 줄였습니다.
- `자막/장면 촘촘히 나누기` 기본값을 2.0초로 변경했습니다.
- 이미지 1장이 자막 2칸 이상에 걸려도 확대/위로/아래로 움직임이 처음부터 다시 시작되지 않습니다.
- 움직임은 해당 이미지 장면의 전체 시작~끝 시간 기준으로 끝까지 이어집니다.
- `자막 2칸당 이미지 1장` 설정을 사용해도 움직임이 자연스럽게 연결됩니다.

## v2.0 최종 수정 내용
- 참고 YouTube Shorts URL 입력칸을 추가했습니다.
- 참고 영상(mp4) 업로드 후 길이/장면 수를 분석하는 기능을 추가했습니다.
- 새 주제를 입력하면 참고 스타일과 비슷한 분량의 쇼츠 대본을 생성합니다.
- 생성된 대본을 자막칸으로 바로 적용할 수 있습니다.
- AI 이미지 스타일을 `인물`, `풍경`, `사람들과` 3개로 정리했습니다.
- 기본 이미지 프롬프트에서 `사람 없음`을 제거했습니다.
- 썸네일 스타일을 `만화형`, `실사형`으로 선택할 수 있습니다.
- 미리보기 캔버스를 270x480으로 줄이고 화면 표시도 더 작게 조정했습니다.
- 출력 비율을 `세로 9:16`, `가로 16:9`로 선택할 수 있습니다.


## v2.2 수정 내용
- 시작할 때 샘플 자막을 자동으로 넣지 않도록 수정했습니다.
- 새 대본 적용 시 기존 샘플/이전 장면을 먼저 지웁니다.
- 대본 생성 결과에서 별표, 느낌표, 쉼표, 따옴표, 마침표 등 특수기호를 자동 제거합니다.

## v2.3 수정 내용

- 표식을 누르면 API 키 발급 안내 박스가 작게 열립니다.
- `API 키 발급 화면 열기` 버튼으로 OpenAI API 키 페이지를 바로 열 수 있습니다.

- 참고영상 문구를 영상 업로드 길이 기준 대본 생성 흐름에 맞게 정리했습니다.


## v2.4 수정 내용

- 첨부한 영상 길이를 자동으로 읽어서 대본 길이와 장면 수 기준으로 사용합니다.
- `이 영상 길이로 대본 맞추기` 버튼을 추가했습니다.
- 음성 파일 선택 시 `음성 첨부됨` 상태가 표시되고 영상 생성 때 오디오 트랙으로 들어가도록 수정했습니다.
- 영상 생성 중 오디오 재생 이벤트가 미리보기 모드로 다시 돌아가던 문제를 막았습니다.
- 남성 음성 샘플은 브라우저에 남성 한국어 음성이 없을 때도 낮은 피치로 강하게 보정되도록 수정했습니다.

- 썸네일은 첫 장면 이미지와 별도로 생성되고, 별도 다운로드가 가능합니다.


## v2.7 긴급 수정 내용
- 업로드한 mp4 영상이 캔버스 미리보기에도 같이 나오도록 수정했습니다.
- 미리보기 재생 버튼을 누르면 첨부 영상도 같이 재생됩니다.
- 별도 음성 파일이 없으면 첨부 영상 속 원본 음성이 영상 생성에 들어갑니다.
- 음성 파일이 따로 있으면 음성 파일 기준으로 첨부 영상 화면이 싱크됩니다.
- 세로 영상은 가로폭을 화면에 맞추고 세로 비율은 그대로 유지합니다.


## v2.8 최종 수정 내용
- 왼쪽 영상 업로드 영역의 영상 미리보기를 제거했습니다.
- 왼쪽은 `동영상 첨부됨`, 파일명, 영상 길이만 표시합니다.
- 실제 영상 미리보기는 가운데 캔버스 1개에서만 재생됩니다.
- 음성 파일이 따로 있으면 영상은 무음으로 캔버스에 표시되고, 음성 파일이 없으면 영상 속 소리를 사용합니다.
- 중복 미리보기 렌더링을 줄여 브라우저 버벅임을 낮췄습니다.


## v2.9 수정 내용
- 썸네일 설정 영역을 왼쪽 입력창 위쪽으로 이동했습니다.
- 썸네일은 글자 전용으로 생성되며 첫 장면 이미지와 분리됩니다.
- 메인 제목/부제목이 좌우로 넘지 않도록 자동 줄바꿈과 자동 글자 축소를 적용했습니다.
- 메인 글자 크기, 부제 글자 크기, 글자 위쪽 위치를 조절할 수 있습니다.
- 세로 9:16 썸네일 상단 안전 영역에 맞춰 요즘 쇼츠형 큰 글자 스타일로 생성됩니다.

- 썸네일 생성 방식이 아니라 영상 전체 위쪽에 고정되는 `상단 고정 썸네일 글자` 방식으로 변경했습니다.
- 왼쪽 위 설정칸에서 메인 제목, 부제목, 글씨체, 크기, 위쪽 위치를 조절하면 가운데 미리보기에 바로 반영됩니다.
- 글자는 좌우 칸을 넘지 않도록 자동 줄바꿈/자동 축소됩니다.
- 기본은 전체 영상에 고정 표시됩니다.
- 오른쪽 장면 카드에서 `상단 고정글자 표시`를 끄면 해당 장면만 중간중간 빼고 끼울 수 있습니다.
- 별도 썸네일 이미지 생성 버튼 없이 영상 레이어 위에 바로 올라갑니다.

- 메인제목/부제목 색상을 각각 선택할 수 있게 했습니다.

- 글자는 좌우 안전영역을 넘지 않도록 자동 줄바꿈/자동 축소됩니다.


## v3.4 긴급 수정 내용
- MP4 변환 기능은 제외하고, 우선 WEBM 영상 다운로드가 확실히 되도록 수정했습니다.
- 영상만 첨부해도 다운로드용 영상 생성이 가능합니다.
- 영상 생성 완료 후 자동 다운로드를 1회 시도합니다.
- 자동 다운로드가 막히면 `영상 다운로드 WEBM` 버튼으로 직접 저장할 수 있습니다.
- 녹화 종료 누락 방지를 위해 강제 종료 타이머와 수동 다운로드 링크를 추가했습니다.
- 음성 파일이 있으면 음성 파일 소리를 사용하고, 없으면 첨부 영상 원본 소리를 사용합니다.


## v3.5 소비자용 다운로드 / Ondoku 수정 내용
- 상단 `프로젝트 저장`을 `작업 저장(JSON)`으로 변경했습니다.
- 가운데 `영상 생성` 버튼을 `쇼츠 만들기`로 변경했습니다.
- 쇼츠 생성 중에는 버튼이 `쇼츠 생성중 %`로 바뀌고 클릭이 잠깁니다.
- 생성 완료 후 같은 버튼이 `영상 다운로드`로 바뀝니다.
- 다운로드 전에는 다운로드 링크가 비활성 상태로 보입니다.
- 영상은 현재 브라우저 안정성을 위해 WEBM으로 저장됩니다.
- Ondoku 바로가기 버튼을 추가했습니다.
- 대본 생성 또는 AI 자막 생성 후 음성 추출용 텍스트칸에 자동 입력됩니다.
- `복사하고 Ondoku 열기` 버튼으로 텍스트 복사 후 Ondoku 페이지를 열 수 있습니다.

## v3.1 유튜브 업로드 수정
- 기존 WEBM 다운로드 때문에 유튜브 업로드가 안 되는 문제를 줄였습니다.
- 브라우저가 MP4 녹화를 지원하면 바로 MP4로 저장합니다.
- MP4 녹화가 안 되는 브라우저에서는 WEBM 생성 후 ffmpeg.wasm으로 MP4 자동 변환을 시도합니다.
- MP4 변환 엔진은 처음 1회 CDN에서 불러오므로 인터넷 연결이 필요합니다.
- 변환 실패 시 WEBM으로 저장되며, 이 경우 캡컷/브루에서 MP4로 다시 내보내면 됩니다.


## v3.4 수정 내용
- 쇼츠 만들기 클릭 시 0%에서 멈추지 않도록 로딩/재생/렌더링을 동시에 시작합니다.
- 자막은 있는데 장면이 없으면 쇼츠 생성 전에 장면을 자동 생성합니다.
- 업로드 영상만 있어도 쇼츠 생성이 진행됩니다.
- WEBM 생성 후 MP4 변환이 끝나면 같은 버튼이 영상 다운로드로 바뀝니다.
- 다운로드는 자동으로 바로 실행되지 않고 사용자가 직접 누르게 변경했습니다.
